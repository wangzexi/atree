# atree MVP 行为验收（BDD）

> 目标：把“行为”固定下来，减少回归时 DOM 与实现细节耦合。
> 这些用例用于后续版本迭代前的快速核验，不替代固定化的单元测试。

## 1. 术语

- **目录节点**：根目录下可在左侧树中浏览的文件夹。
- **关键节点**：存在 `.agents/atree/meta.yaml`、会话目录或自动化消息的目录节点。
- **会话**：目录下的 `.agents/atree/sessions/<session-id>/`，其中 `session.jsonl` 是原始记录。
- **自动化**：会话目录中 `schedule.json` 记录的 `at` / `cron` 自动化消息。

## 2. 行为场景（BDD）

### Scenario: 关键节点优先展示，普通目录可展开
- Given 打开根目录
- When 节点扫描完成
- Then 左侧优先展示关键节点
- And 无会话的普通目录可通过父节点展开查看
- And 普通目录不会污染右侧会话 tab 计算

### Scenario: 根目录切换不污染会话组
- Given 已有两个根目录
- When 切换到另一个根目录
- Then 仅保留当前根目录对应的节点树
- And 旧根目录会话不再进入右侧 tab 计算

### Scenario: 节点切换会重算会话分组
- Given 节点 A、B 都有会话
- When 从 A 切换到 B
- Then 右侧只展示 B 的会话 tab
- And A 的会话在右侧不再展示
- When 再切回 A
- Then 恢复 A 最近激活会话（若有）

### Scenario: 纯目录节点默认收敛
- Given 某节点下只有子目录无会话
- When 节点未展开
- Then 该子树在左侧默认收敛
- When 展开该节点
- Then 显示子目录结构
- And 收起后恢复默认隐藏

### Scenario: 新建会话与首条消息
- Given 选中任一 atree 节点
- When 发送第一条消息
- Then 即时新增会话 tab（无需刷新）
- And 该会话可直接继续输入

### Scenario: 会话分组状态恢复
- Given 在节点下已打开多个会话，存在激活会话
- When 切到其他节点再返回
- Then 仅恢复目标节点的会话分组
- And 恢复该节点最后激活会话

### Scenario: 归档会话双确认
- Given 当前会话有自动化任务
- When 点击归档
- Then 进入确认状态
- And 二次确认后进入归档
- And 当前 tab 清空该会话显示
- And 自动化任务一起清理

### Scenario: 归档恢复
- Given 存在归档会话
- When 从归档列表恢复会话
- Then 恢复到当前节点 tab
- And 不再展示在归档列表

### Scenario: 自动化是互斥的
- Given 一个会话已有一条自动化
- When 再设置新的自动化
- Then 系统拒绝或提示先清理旧任务
- And 会话最终只保留一条自动化

### Scenario: at 与 cron 的显示策略一致
- Given 会话自动化为 `at`
- When 触发时间在 12 小时内
- Then 展示相对时间
- When 超过 12 小时
- Then 展示绝对时间

### Scenario: 自动化执行后立即刷新
- Given 自动化会话触发 `schedule.ran`
- When 事件处理完成
- Then 会话头与排序、树节点图标即时更新
- And 不需要手工刷新页面

### Scenario: 仅会话来源于当前节点与当前状态
- Given 会话不在当前节点或已归档
- When 计算节点图标与 tab 列表
- Then 不出现跨节点或归档状态重复

### Scenario: 调用链路单一路径
- Given 新建会话、恢复/归档会话、切换 tab
- When 执行路径
- Then 每个行为只走一条明确入口（`create` / `promoteDraft` / `archive`）
- And 不出现并行或重复提交导致状态抖动

## 3. 代码整理验收

### Scenario: 调度事件解析路径单一
- Given 后端推送 `serverSDK.event` 事件
- When 触发 schedule 事件监听
- Then 只基于 `event.details` 提取 `type`/`properties`
- And 不再保留旧的多形态兼容分支

### Scenario: BDD 用例文档可回放
- Given `docs/acceptance/atree-mvp-bdd.md` 存在
- When 进入下一次版本收尾
- Then 人工只按场景与手工清单即可复核
- And 不依赖 Playwright 精确元素断言
