# atree-ng MVP 行为验收（BDD）

> 目标：保留 MVP 核心行为，减少对 DOM 细节的依赖。
> 下面场景用于后续版本快速回归；不是单元素 Playwright 用例。

## 术语

- **节点（Node）**：包含 `.agents/atree.yaml` 的目录。
- **会话（Session）**：该节点下 `.agents/sessions/*.jsonl` 的聊天会话。
- **自动化（Schedule）**：会话的 `cron` / `at` 任务。

## 1. 节点树与树切换

### Scenario: 仅展示 atree 节点
- Given 启动时传入一个有效根目录
- When 目录扫描结束
- Then 左侧仅展示包含 `/.agents/atree.yaml` 的节点
- And 普通目录（无 atree 节点身份）不进入会话树

### Scenario: 节点树根目录切换只剩一个工作根
- Given 存在多个已打开的工作根
- When 用户执行“切换根目录”选择其他根
- Then 仅当前根目录对应的节点树保留
- And 非当前根目录中的会话上下文不应再参与渲染

### Scenario: 切换节点后会话组隔离
- Given 节点 A 有会话，节点 B 有会话
- When 用户从 A 切到 B
- Then 右侧 Tab 仅展示 B 下会话组
- And A 下会话在右侧不可见
- When 再切回 A
- Then 恢复 A 的会话组与最近激活会话

### Scenario: 子树默认收敛展示
- Given 某节点仅含无会话子目录
- When 节点未展开时
- Then 不在左侧默认显示该无会话分支
- When 点击展开
- Then 显示可选的子目录分支
- And 展开状态可回收

### Scenario: 节点会话优先排序
- Given 一个节点下既有带自动化也有普通会话
- When 左侧节点会话图标展示
- Then 自动化会话靠前
- And 自动化会话按下一次执行时间从近到远排序
- And 非自动化会话按最近更新排序

## 2. 会话生命周期

### Scenario: 新建会话
- Given 用户在某节点右侧有会话入口
- When 发送第一条消息
- Then 在该节点会话列表新增会话记录
- And `.agents/sessions/<session-id>.jsonl` 被创建或追加
- And 会话元数据在 `.agents/atree.yaml` 中可追踪

### Scenario: 会话组状态恢复
- Given 用户在节点 A 下打开了会话 A1、A2，且当前激活 A2
- When 回到首页再重新选择 A
- Then 右侧 Tab 仍然以 A2 为当前会话
- And 不加载 B 节点的历史会话

### Scenario: 关闭（归档）二次确认
- Given 当前会话带有自动化
- When 点击归档按钮
- Then 进入确认状态（仍停留在当前列表）
- When 再次确认归档
- Then 该会话转入归档，不再在当前 Tab 列表出现
- And 自动化随会话一并清理

### Scenario: 归档恢复
- Given 存在归档会话
- When 从归档列表恢复该会话
- Then 会话立即回到当前节点 Tab 组
- And 不再出现在归档列表

## 3. 自动化（Schedule）行为

### Scenario: 支持 once 与 recurring
- Given 会话无自动化
- When 用户通过会话内命令设置一次性时间点
- Then 记录 `at`（一次性）
- When 设置周期表达式
- Then 记录 `cron`（周期性）

### Scenario: 每会话最多一条自动化
- Given 会话已存在一条自动化
- When 再次设置新自动化
- Then 拒绝并提示“先取消旧自动化”
- And 不新增第二条记录

### Scenario: 自动化变化立即生效
- Given 会话已有自动化
- When 调用 create/delete 并返回成功
- Then 会话头与左侧图标优先级即时更新
- And 无需刷新页面

### Scenario: 一次性自动化只触发一次
- Given 一个 `at` 调度处于未来
- When 到达触发点
- Then 该条消息按会话发送一次
- And 发送完成后不会再次触发

### Scenario: 自动化执行时间展示策略
- Given 自动化在 12 小时内
- Then 使用相对时间
- When 超过 12 小时
- Then 使用绝对时间

### Scenario: 自动化排序与节点头部图标
- Given 节点内有多个自动化会话
- Then 列表按执行先后排序
- And 节点行首图标来自最早执行自动化会话

## 4. 稳定性与恢复

### Scenario: 重启后从会话元数据恢复
- Given 有已归档与未归档会话
- When 重启应用并切回原根目录
- Then 会话列表按元数据恢复到对应节点
- And 当前激活节点仍只含该节点可见会话

### Scenario: SSE 非自动化事件不会误刷新
- Given 正常监听事件流
- When 收到非 schedule 事件
- Then 不触发 schedule 重拉与排序抖动

## 5. 视觉回归的手工验收（至少 10 次）

#### 轮次 1：初次启动
- Given 全新启动且仅有一个根目录
- When 查看左侧树和右侧内容
- Then 显示根节点 + “选择会话/新建会话”空态

#### 轮次 2：重复点击当前节点
- Given 已进入某节点并打开多个会话
- When 快速连续点击该节点
- Then 不出现重复会话组、Tab 不闪烁

#### 轮次 3：跨节点切换
- Given A/B 均有会话
- When 循环切换 A 与 B
- Then 仅显示当前节点会话
- And 会话组切换无残留

#### 轮次 4：展开树层级
- Given 子节点下有会话与无会话目录
- When 展开/收起
- Then 会话节点始终可见
- And 纯目录节点按展开状态展示

#### 轮次 5：新建会话第一条消息
- Given 当前处于某节点
- When 发送第一条消息
- Then 新会话立刻可见并可继续输入

#### 轮次 6：归档确认
- Given 会话有自动化
- When 执行归档两次确认
- Then 会话进入归档且当前列表移除

#### 轮次 7：归档恢复
- Given 归档中有会话
- When 恢复
- Then 恢复到对应节点的 Tab，并从归档移除

#### 轮次 8：设置自动化（at + cron）
- Given 任一普通会话
- When 设置 `at` 后设置 `cron`
- Then `at` 提示需先取消旧任务
- And cron 创建后头部与节点图标更新

#### 轮次 9：自动化执行后同步
- Given 自动化会话被触发
- When 后端返回 `schedule.ran`
- Then 头部时间和排序即时刷新

#### 轮次 10：消息即时显示与滚动
- Given 发送一条消息
- When 流式返回中
- Then 用户消息立即可见
- And 滚动条行为稳定，不出现遮挡

#### 轮次 11：恢复未归档状态后稳定性
- Given 频繁切换会话后关闭应用
- When 重开并回到同一节点
- Then Tab 分组与归档恢复一致

#### 轮次 12：非 atree 目录干扰
- Given 有普通目录混入同级目录
- When 在树里选择会话入口
- Then 不会展示非 atree 目录会话上下文

## 6. 会话树与目录切换边界

### Scenario: 会话归属不重复
- Given 一个会话同时被归档和打开
- When 应用重建状态
- Then 会话只出现在一个列表中
- And 不会同时存在于 tab 区和归档列表

### Scenario: 无会话目录点击行为
- Given 当前在有会话的目录
- When 切到无会话目录（仅可展开/显示普通目录）
- Then 右侧显示目录自身空态，不带上一个目录的会话痕迹

### Scenario: 路由切换与树选中一致
- Given URL 指向某目录+会话
- When 页面刷新或回退
- Then 左侧树高亮与该目录一致
- And tab 组仅显示该目录可见会话

## 7. 自动化一致性与元数据

### Scenario: at 与 cron 的互斥执行
- Given 会话存在一次性 `at`
- When 设置 `cron` 时不先取消旧任务
- Then 返回替换提示或拒绝并保持单任务约束
- And 最终会话仅保留一条任务

### Scenario: 自动化排序与刷新
- Given 节点下有多条定时会话且执行时间不同
- When 任何一次触发完成
- Then 列表按下一次执行时间自动重排
- And 节点头部图标跟随最早触发任务

## 9. 代码边界化与简化验收

> 这组场景不是功能新增，而是防止“兼容分支化”继续堆叠。

### Scenario: 新建会话路径保持单一实现
- Given `isNewSession === true`
- When 完成会话初始化分支
- Then 仅执行“新建会话 + 更新元数据 + 写入 store + 打开 tab”这条唯一路径
- And 不允许存在可选/回退分支导致行为二义性

### Scenario: Tab 新建不经过多级包装
- Given 用户发起新会话
- When 当前 URL 无 draft 参数
- Then 直接走 `tabs.addSessionTab`
- When 存在 draft 参数
- Then 直接走 `tabs.promoteDraft`
- And 不允许出现既 promote 又 add 的重复调用

### Scenario: 会话可见性来源仅为 tab store
- Given 某目录有未归档的活跃会话
- When 计算会话图标占用
- Then 仅按 `tabs.store` 与 `serverSync.child(dir).session` 的最终状态判断
- And 不依赖中间兜底数组或未使用的兼容路径

### Scenario: 调度头部与列表一致
- Given 一个会话被设为自动化
- When schedule 变更成功返回
- Then 显示顺序与 icon 优先级立即收敛
- And 不依赖额外适配层进行“二次排序”
